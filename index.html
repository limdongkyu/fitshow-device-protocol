跑步机蓝牙协议
v1.1
协议格式标准

蓝牙通讯协议采用帧进行通讯，每一帧采用了起始头、校验码、终止尾的方式进行数据校验，基本数据格式如下：

0x02 CMD DATA FCS 0x03
起始码 指令 数据 校验码 终止码
*注：校验码是除去指令头尾，各个字节进行异或后的值
设备返回的数据格式：
0x02 CMD DATA FCS 0x03
起始码 指令 数据 校验码 终止码

发送数据后设备将按上述格式返回，意味着返回数据包含发送时的指令，在程序中可以直接依次指令做出不同的处理

发送与接收数据采用小段对齐方式。即数据低字节在前、高字节在后。

数据类型：B：字节; W：整形; L：长整型; N：变长

说明：为将来扩展考虑规定：当电子表收到未在本协议中出现的指令时（指令正确）应按格式只返回指令（不包含数据部分）

车表设备蓝牙协议
v1.0
（健⾝车、椭圆机、划船机、登⼭机、踏步机）
协议格式标准

蓝⽛通讯使⽤帧⽅式通讯，每⼀帧采⽤了起始头、校验码、终⽌尾的⽅式进 ⾏数据校验，帧间隔最⼤ 250ms，在正确接收到帧返回数据后，⽴即进⾏下⼀帧 通讯。数据格式如下：

0x02 CMD DATA FCS 0x03
起始码 指令 数据 校验码 终止码

协议：baud, N, 8, 1　 (⽆校验，8位数据，1位停⽌)

波特率：4800、9600、19200、38400、115200 ⾃适应

起始码：固定的1字节数据，为⼗六进制：0x02；

指令：1字节的指令，或2字节包含⼦指令；

数据：写⼊或读取的数据，长度依指令不同⽽不同；

校验码：指令/数据 每字节进⾏异或后的值（1字节）；

终⽌码：固定的1字节数据，为⼗六进制：0x03。

*数据采⽤⼩端对齐⽅式，即低字节在前、⾼字节在后。数据类型： B:字节　W:整型　L:长整型　N:多字节　(未指定类型数据均为字节型)

*设备收到未在本协议中出现的指令（格式符合标准并验证成功）时应返回包含当前指令的帧，如下所⽰：

START CMD DATA FCS END
0x02 0x7F 0x01 0x02 0x7C 0x03
START CMD FCS END
0x02 0x7F 0x7F 0x03

蓝牙模块通用指令

蓝⽛模块在上电启动后会发送配置指令到设备，设备需按格式返回数据。

1. 配置设备型号（必要）0x50

设备型号是APP识别的依据，因此设备必须按格式实现机型代码数据 返回。否则运动秀将⽆法识别当前蓝⽛模块。

蓝⽛发送的数据：（蓝⾊表头代表蓝⽛发送到设备，下同）

START CMD FCS END
0x02 0x50 0x00 0x50 0x03

设备返回的数据：（绿⾊表头代表设备发送到蓝⽛，下同）

START CMD DATA FCS END
0x02 0x50 0x00 品牌（W） 机型（W） xx 0x03

*品牌、机型代码分别是16位的⽆符号数，品牌代码由运动秀分配后固 定，新机型代码需由制造商通过运动秀官⽹平台申请

设备参数 0x41

APP需要⼀些设备的参数来适配相关操作，因此APP会在连接蓝⽛模块后发 送请求指令以获取设备参数。

1. 设备参数

APP通过此指令获取设备相关参数

START CMD FCS END
0x02 0x41 0x02 0x40 0x03
START CMD DATA FCS END
0x02 0x41 0x02 阻力（B） 坡度（B） 配置（B） 段数（B） xx 0x03
  • 阻⼒：设备⽀持最⼤阻⼒级别，APP将从0开始到此值进⾏调节；
  • 坡度：若设备⽀持坡度则返回⽀持的总坡度值，否则返回0；
  • 段数：⽀持程式模式的段数，APP会转换或过滤相同段数的程式， 以⽅便设备显⽰程式数据，若不限制返回1（APP发送 8 - 48段间的 程式）；返回0则说明设备不⽀持APP启动程式模式。
  • 配置：bit0 (1代表英制单位) bit1 (1代表支持暂停功能；）支持坡度的时候，高4位用于表明设备负坡度范围，不支持负坡度则为0)
2. 累计值

设备感应器累计感应次数，若设备不⽀持此指令则返回的数据不包含DATA部分。

START CMD FCS END
0x02 0x41 0x03 0x42 0x03
START CMD DATA FCS END
0x02 0x41 0x03 累计值（L） xx 0x03
3. 同步设备时间

APP连接蓝⽛并获取参数成功后，会发送⼀条时间同步指令，若设备有时间功能，可⽤此指令包含的时间来同设备上的时间。

START CMD DATA FCS END
0x02 0x41 0x04 时间（7） xx 0x03

data字段中数据如下

YEAR MONTH DAY WEEK HOUR MIN. SEC.
21 5 6 4 13 45 0
START CMD FCS END
0x02 0x41 0x04 0x45 0x03
  • *YEAR：为00~99的年份值。
  • *WEEK：0 星期⽇，1~6 星期⼀~星期六。

设备状态 0x42 、设备数据 0x43

APP连接蓝⽛后在待机及运动中，APP会实时获取设备当前状态数据来决定 相应的数据显⽰。

1. 获取设备状态

设备当前状态是APP运⾏的依据，也是APP与设备互通的基础，因此在 APP连接蓝⽛后就会以每秒3次的周期不间断获取设备状态。

START CMD FCS END
0x02 0x42 0x42 0x03
START CMD DATA FCS END
0x02 0x42 状态（B） 数据（N） xx 0x03

状态及数据对应如下

名称 状态 数据
待机 0
启动中 1 启动倒计时（B）秒
运行 2 速度（W） 阻力（B） 频率（W） 心率（B） 瓦特（W） 坡度（B） 段索引（B）
睡眠 20
故障 21 故障码（B）
  • 速度是以 0.01 为单位的16位整数，如：1000，则为 10.00 km/h；
  • 频率对应踏频、步频等，名称与实际设备对应；
  • ⽡特单位为 0.1⽡，如 1000 代表 100.0 ⽡；
  • 段索引仅在程式模式时使⽤，从1开始到程式长度；
  • 设备不需要启动倒计时，状态直接从待机变化到运⾏。
2. 获取运动数据

设备运⾏后通过此指令来获取当前运动量数据。

START CMD FCS END
0x02 0x43 0x01 0x42 0x03
START CMD DATA FCS END
0x02 0x43 0x01 时间（W） 距离（W） 热量（W） 计数（W） xx 0x03
  • 运动量不受倒计模式影响，数据总是从0开始向上计；
  • 时间是运动所⽤时长，以秒为单位；
  • 距离在⼩于32000时，单位为1，否则最⾼位置1同时单位为10；
  • 计数为本次运动开始后感应器感应次数。
  • 距离：如当前距离为30000⽶，则返回 30000 (0x7530)；若运动距离达到 50000⽶时，返回 37768 (0x9388)，值 = 距离除以10后将最⾼位置1。也 可以⼀开始将最⾼位置1来告诉APP距离都是以为10⽶为单位。
3. 获取运动信息

APP连接蓝⽛后，若由设备端启动，则APP会发送当前指令来获得当前 运动信息，根据模式进⾏不同显⽰，同时也判断是否与当前⽤户冲突。

START CMD FCS END
0x02 0x43 0x02 0x41 0x03
START CMD DATA FCS END
0x02 0x43 0x02 用户ID(L) 运动ID(L) 模式(B) 段数(B) 目标(W) xx 0x03
  • ⽤户ID、运动ID是设备控制指令写⼊的值，在此回传；
  • 模式参见设备控制部分的设备启动指令处；
  • 段数为程式模式下对应的程式的总段数，其它模式为0；
  • ⽬标是倒计模式或程式的⽬标总值，其它模式为0。
  • ⽬标为距离时，遵守⾼位置1标准，见上述。
4. 获取程式数据

由设备开始程式模式运动时，APP会发送此指令来获取程式数据。

START CMD DATA FCS END
0x02 0x43 0x03 0x41 0x03
START CMD DATA FCS END
0x02 0x43 0x02 用户ID(L) 运动ID(L) 模式(B) 段数(B) 目标(W) xx 0x03
